# Contributing to DCPM

First off, thank you for considering contributing to DCPM! It's people like you that make the open-source community such a great place to learn, inspire, and create.

## Code of Conduct
By participating in this project, you agree to maintain a respectful and professional environment.

## How Can I Contribute?

### Reporting Bugs
* Check the [Issues](https://github.com/dragusheen-studio/dcpm/issues) to see if the bug has already been reported.
* If not, create a new issue. Include:
    * A clear and descriptive title.
    * Steps to reproduce the problem.
    * Your OS and C++ compiler version.

### Suggesting Enhancements
* Open an issue with the tag `enhancement`.
* Explain why this feature would be useful to most users.

### Pull Requests
1. Fork the repository.
2. Create your feature branch (`git checkout -b feature/AmazingFeature`).
3. Commit your changes (`git commit -m 'Add some AmazingFeature'`).
4. Push to the branch (`git push origin feature/AmazingFeature`).
5. Open a Pull Request.

## Style Guidelines
* **Python**: Follow PEP 8 (handled by our `.editorconfig`).
* **C++ Templates**: Use the default formatting provided by the `.clang-format` generated during `dcpm init`.
* **Documentation**: Keep the README and Wiki updated for any changes in command behavior.

## Questions?
If you have any questions, feel free to open an issue for discussion.
